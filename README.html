GiTrends 1.0

GiTrends is a RESTful API that helps get information about trending repositories from Github

Context :

GiTrends was developed in the context of my application for the "Data Engineer Intership" poistion at Gemography

It was developed using Spring Boot and Jsoup. It is secured using JWT.

Usage :

  • Authentication :

    The first step is to generate a JWT token that will help authenticate every request to the server. You can do so by sending a POST request to the following route:

    localhost:8081/api/v1/authenticate

    with the default user in the request body:

    {
    "username":"foo",
    "password":"bar"
    }

  • Routes :

    After authenticating, the JWT should be added to the Authorization header after "Bearer " with every request.
    1. List the languages used by the 25 trending repositories:
      localhost:8081/api/v1/repositories/trending/languages
    2. Get the number of trending repositories using a language (case insensitive path variable):
      localhost:8081/api/v1/repositories/trending/languages/{language}/count
    3. List the repositories using a language (case insensitive path variable):
      localhost:8081/api/v1/repositories/trending/languages/{language}
    4. Language popularity over the 25 trending repositories:
      localhost:8081/api/v1/repositories/trending/languages/popularity

  • Responses :

    Responses for valid requests take the following form :

    {
    "message":"an explanatory message",
    "value":"the value or values requested"
    }

  • The missing features:

    In the first place, I expected the Github API to expose an endpoint of trending repositories but I didn't find any. So i decided to scrape the trending repositories from Github website instead of the API. on this web page no information is provided about frameworks, that is why I calculated programming languages popularity instead.
